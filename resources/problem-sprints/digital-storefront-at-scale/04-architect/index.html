The Problem

Every acquisition is a months-long digital nightmare. The existing network is inconsistent. The team is tiny.

Before — Current State

  • Months to set up each practice's digital storefront
  • Days just to recover passwords and account access
  • Every website is custom — no templates, no consistency
  • No standardized GMB, social, or lead gen across practices
  • Practices view HQ marketing as a nuisance
  • No ongoing curation — storefronts decay after setup
  • Staff changes go undetected across 7+ digital touchpoints
  • Zero executive visibility into network digital health
  • 10-person team manually managing 260+ locations

After — Practice Concierge

  • Under 2 weeks from acquisition to live storefront
  • Pre-populated profiles from public data — no password chasing
  • Specialty-specific templates: GP, Perio, Full Arch, Pediatric
  • GMB, social, CRM auto-configured and synced from day one
  • Practices talk to their concierge — 24/7, 5-minute interactions
  • 365-day automated curation with human QC on exceptions
  • One change → automatic cascade across all touchpoints
  • Real-time network health dashboard with automated scoring
  • Same 10 people managing 300+ practices via automation

Sprint Questions

Can we reduce practice setup from months to under 2 weeks? High Risk
Multi-layered bottleneck: asset access, design decisions, GMB setup, CRM integration. Requires both technical (templates, automation) and behavioral (practices providing info) solutions.
Can a ~10-person team maintain quality across 300+ storefronts? High Risk
26+ practices per person. Quality control must be programmatic — templates, brand guardrails, automated checks — with humans only on exceptions.
Will practices engage with an agent instead of ignoring it? High Risk
Practices currently view HQ as a nuisance. The hypothesis: always-available, immediately valuable agent gets more engagement than calls and emails.
Can we detect practice-level changes proactively? Medium Risk
Changes come through random emails or not at all. Solution: agent as reporting channel + iSolved HR integration for automatic detection.
Can we provide exec visibility without reporting burden? Medium Risk
Dashboard must be fully automated — pulling from live sources. If it requires manual data entry, it won't be maintained.

User Journey

How a practice goes from current state to fully managed digital storefront.

Step 1 — Before Contact

Discovery & Pre-Population

System silently harvests public data for every practice — websites, GMB, social profiles, reviews, directories. Builds a Practice Profile Draft with confidence scores.

The "we already know about you" moment. Quality of pre-population determines first-impression impact. Uses LLM-assisted extraction to parse any website structure.
Step 2 — Invitation

Enrollment Invitation

Batches of 20-30 practices receive personalized invitations. "Your SGA Marketing Concierge already knows about your practice — meet them and get upgraded in 15 minutes."

Must feel like a gift, not a mandate. The preview data demonstrates value before asking for anything. The 15-minute commitment is honest and low-friction.
Step 3 — Verification

Concierge Welcome & Data Verification

Concierge presents what it found. Practice corrects in 3-4 minutes: "Dr. Miller left, add Sarah the new hygienist." Every correction triggers downstream cascades automatically.

Tone is Sage + Caregiver — knowledgeable, warm, never bureaucratic. Corrections are handled gracefully: "Got it — I'll remove Dr. Miller from everything. Do you have a photo of Sarah?"
Step 4 — Generation

Storefront Generation & Template Selection

Concierge presents 2-3 specialty-optimized templates populated with real practice data. Recommends with reasoning: "Perio sites convert 23% better with referral-optimized layouts."

Guardrailed decisions — every option is SGA-approved. If no response in 3 weeks, auto-deploys the recommendation. The 10% with strong opinions get a customization path within guardrails.
Step 5 — Review

Full Storefront Review & Approval

Complete package presented: website, GMB, social profiles, lead capture. HQ creative reviews in parallel (24-hour QC window). Practice approves and everything deploys.

99% pass QC without issue. The practice feels ownership while SGA maintains quality control. Real-time changes handled in-conversation.
Step 6 — Go Live

Activation

Website deployed, GMB updated, social profiles refreshed, forms connected, auto-responses enabled. Concierge sends 1-week follow-up with traffic and lead data.

First data delivery reinforces value and builds the check-in habit: "Your new site had X visitors this week. Y appointment requests. Here's how you compare to other practices."
Step 7 — The Moment of Truth

Ongoing Change Management

A doctor leaves. Office manager tells the concierge. It responds with a complete action plan: website, GMB, social, ads, signage — and schedules everything for the departure date.

This is where long-term value is proven. The proactive suggestion list ("here's what I'll handle + here's what you need to handle locally") demonstrates the agent's opinion and thoroughness.
Step 8 — Safety Net

Blitz Deploy (Non-Engaged Practices)

After 30 days, ~50 practices haven't engaged. Default storefronts auto-deploy from scraped data + conservative templates. "I went ahead and launched your upgrade — customize anytime."

100% network coverage regardless of engagement. Dashboard shows these as "auto-deployed" vs "practice-customized" for visibility into engagement tiers.

System Architecture

Six components connected by an event-driven orchestration layer.

■

Practice Data Hub

Single source of truth. One canonical record per practice. Change here → propagates everywhere. PostgreSQL + Redis cache.

■

Data Harvester

Pre-populates practice profiles from public data. LLM-assisted extraction from any website structure. Runs before first contact.

■

Concierge Agent

Per-practice AI concierge. Enrollment, change management, content approval, Q&A. Opinionated — recommends with reasoning.

■

Template Engine

5 specialty-specific website templates. Generate from structured data. Batch update all sites using a template in one operation.

■

Channel Sync Engine

Event-driven sync to GMB, social, directories, CRM. Change fires immediately — not nightly batch. Retry with backoff.

■

Network Dashboard

Executive + HQ views. Health scores, alerts, QC queue, batch operations. Fully automated — zero manual data entry.

Key Decisions

Architecture Decision Records — the reasoning behind critical choices.

ADR-1: One Agent Per Practice (not shared)

Decision: Deploy one logical agent instance per practice — shared model, practice-specific context.

Why: Per-practice agents feel personal and eliminate context-switching errors. Cost is minimal — the LLM is shared, only the system prompt and context window differ. At 260+ practices, the overhead is context storage, not compute.

Trade-off: Practice data must be efficiently loadable into each agent's context on every interaction.

ADR-2: Generated Sites (not CMS-managed)

Decision: Generate static/JAMstack sites from templates + data, not CMS-managed dynamic sites.

Why: 260 WordPress instances is a maintenance nightmare. Generated sites are faster, cheaper, more secure, and easier to batch-update. One template change re-generates all sites. The Template Engine owns templates; the Data Hub owns data; output is deployable.

Trade-off: Content updates require re-generation. Pipeline must produce live updates within minutes.

ADR-3: Event-Driven Sync (not nightly batch)

Decision: Event-driven synchronization with batch as fallback safety net.

Why: "Dr. Johnson left" can't wait for a nightly batch — the website shows a departed doctor for up to 24 hours. Events fire within minutes. Nightly batch catches anything the event system misses.

Trade-off: Requires a reliable message queue and graceful API rate limit handling.

ADR-4: Blitz Deploy Safety Net (30-day auto-deploy)

Decision: Auto-deploy default storefronts for non-engaged practices after 30 days.

Why: A clean generic storefront from scraped data is better than a broken or nonexistent one. Ensures 100% network coverage regardless of practice engagement. The concierge continues inviting these practices to customize.

Trade-off: Blitz sites may have inaccuracies. Flagged as "auto-deployed — unverified" in the dashboard.

ADR-5: Opinionated Agent with Reasoning

Decision: The Concierge always recommends with reasoning, never just presents options.

Why: SGA's brand is Sage + Caregiver. An agent that says "here are 5 options" adds no value. An agent that says "I recommend Template A because perio sites convert 23% better with referral-optimized layouts" reduces decision fatigue and demonstrates expertise. This is what "forceful opinion" means.

Trade-off: Knowledge base must include conversion data and industry benchmarks. Wrong recommendations erode trust permanently.
