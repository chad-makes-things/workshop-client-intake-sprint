## 10. Blog Article On-Page Template

All blog articles follow this on-page structure:

| Element | Specification |
|---------|--------------|
| **URL slug pattern** | /blog/[keyword-slug] |
| **Title tag formula** | [Primary Keyword] - [Value Proposition or Year] | SGA Dental |
| **Meta description formula** | [Direct answer or hook in 1 sentence]. [What the article covers]. [CTA or authority signal]. |
| **H1** | Match the article title (include primary keyword, naturally phrased) |
| **H2 structure** | 4-8 H2 sections covering the topic comprehensively; include at least 1 FAQ H2 |
| **H3 structure** | Sub-sections under H2s as needed for depth and scannability |
| **Author attribution** | Named author with credentials and photo. For clinical content: Dr. Olson, Dr. Pikos, or Dr. Ellingson. For business content: Lamonte Jensen. |
| **Publication date** | Visible on page, included in Article schema |
| **Internal links** | Minimum 3 internal links per article: 1 to pillar page, 1 to sibling article in same cluster, 1 to relevant landing page |
| **FAQ section** | Every article ends with 3-5 FAQ questions formatted for FAQ schema markup |
| **CTA** | Every article includes at least 1 contextual CTA (in-content) and 1 end-of-article CTA |
| **Schema** | Article, FAQPage, Person (author) |
| **Word count** | Pillar pages: 3,000-5,000 words. Standard articles: 1,500-2,500 words. |
| **Image requirements** | Minimum 2 images per article. Custom graphics or photos preferred over stock. All images require descriptive alt text with target keyword where natural. |

## 12. Technical On-Page Requirements (All Pages)

| Requirement | Specification |
|-------------|--------------|
| **Canonical tags** | Self-referencing canonical on every page |
| **Hreflang** | Not required (English-only, US market) |
| **Open Graph tags** | og:title, og:description, og:image, og:url on all pages |
| **Twitter Card** | summary_large_image for blog articles; summary for landing pages |
| **Mobile viewport** | `<meta name="viewport" content="width=device-width, initial-scale=1">` |
| **Page speed** | Target LCP < 2.5s, FID < 100ms, CLS < 0.1 |
| **Heading hierarchy** | Single H1 per page, H2s for major sections, H3s for sub-sections. No skipped levels. |
| **URL structure** | Lowercase, hyphens (not underscores), no trailing slashes, no query parameters for canonical content |
| **Internal link anchor text** | Descriptive, keyword-relevant anchor text. Avoid "click here" or "learn more" as standalone anchors. |
| **External links** | Open in new tab. Add rel="noopener" for security. Link to authoritative sources (ADA, Becker's, PubMed) to support E-E-A-T. |
| **Breadcrumbs** | BreadcrumbList schema on all pages. Format: Home > [Section] > [Page] |
