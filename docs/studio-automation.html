No standardized intake, no centralized DAM, wildly inconsistent brand assets, and a print vendor (Staples) that adds cost and friction at every order.

Decision: Solution A + Email Channel

The Conversational Studio Agent with email as a third channel (studio@sgadental.com alongside Teams and Slack) was selected unanimously. The sprint recommendation was Solution A; the user refinement added email coverage for universal reach.

  • Works for all 160+ locations without requiring new tool adoption
  • Email is the universal fallback — every practice manager has it
  • Agent handles the full triage and production pipeline autonomously
  • CD review stays as brand quality gate only — not content production
  • Medium challenge is conversational, not a blocking form field
  • Phase 2 proactive triggers (Solution C/E) can layer on top of this foundation

6 Sprint Learnings

Learning 01

Asset baseline is a pre-problem

30–40% of practices lack the brand assets needed for the agent to produce quality output. This must be solved in parallel — not inside the agent build. Phase 1 deliberately uses Gen4 practices where brand kits are complete.

Learning 02

Capacity constraint is friction, not volume

The studio isn't overwhelmed by quantity of requests — it's overwhelmed by the overhead per request: finding assets, interpreting vague briefs, chasing approvals. Removing friction per request is more valuable than adding headcount.

Learning 03

The medium question is the highest-leverage moment

Most practices default to print because nobody ever offered a digital alternative. The medium challenge isn't a rejection — it's "here's something that might work even better." Framed correctly, it converts 30–40% of print requests to digital-only.

Learning 04

CD review should happen after practice approval

Inverting the review sequence (practice confirms content first, CD approves brand quality second) eliminates the biggest source of post-CD revision requests. Practice-driven content changes drop from ~30% to <5% after CD approval.

Learning 05

Canva + agent is the existing nucleus

The studio already uses Canva Enterprise with established brand kits. The agent doesn't replace this — it wraps it with conversational intake and programmatic API access. This dramatically reduces the build scope vs. starting from scratch.

Learning 06

Three brand universes require three Canva workspaces

SGA legacy, Gen4, and MODIS have distinct brand identities. Phase 1 scopes to Gen4 practices only. Multi-workspace agent routing (brand kit selection based on practice platform) is Phase 2 scope.

4 Unresolved Questions

Unresolved · API Scope

Canva Connect API template variable capabilities need validation before committing to the 12-template library. Spike SP-1 must confirm limits before architecture is locked.

Unresolved · Data Privacy

Weave API patient list access: data privacy model and consent requirements for patient email scheduling automation need legal review before Phase 1 deployment.

Unresolved · Review SLA

CD review backup protocol: named backup reviewer and escalation notification logic must be defined before build. The 2-hour SLA is only achievable if backup is defined.

Unresolved · Phase 2 Triggers

Proactive content calendar (Solution C logic) requires mapping which PMS each Gen4 practice uses before any API integration planning. Dentrix, Eaglesoft, and Curve all have different access models.

User Journey

Ashley's April Whitening Special

Ashley is the practice manager at Towne Lake Dental. She needs promotional assets for an April teeth whitening special. Total elapsed time: 52 minutes.

Step 1 · T+0

Ashley emails studio@sgadental.com

Ashley sends a casual email: "We're running a teeth whitening special in April — want to do something for existing patients and maybe new patients too. Need an email header and maybe a flyer?"

No form. No login. Just an email to the studio address. The agent receives it via Microsoft Graph API webhook within seconds.
Step 2 · T+2 min

Agent acknowledges + challenges medium

The agent replies: "Hi Ashley! Got your request for the April whitening special. Before I get started — your Weave patient list shows 847 active patients with email on file. A digital-only package (email header + Weave campaign + TV waiting room slide) would reach them all directly at no print cost. Want me to go that route, or would you still like the flyer?"

Medium challenge logic: agent checks practice's Weave connection, surfaces digital ROI, offers a non-blocking alternative. Tone is "here's something that might work even better" — not an objection.
Step 3 · T+5 min

Brief gathered, assets generated in 45 seconds

Ashley replies: "Existing patients mainly, but new patients too — let's do digital + one flyer for the front desk." Agent extracts brief (audience: existing + new patients, goal: whitening special, month: April, tone: welcoming) and calls Canva Connect API.

Canva generates 3 assets using Towne Lake Dental's brand kit: email header (600×200px), waiting room TV slide (1920×1080), and desk flyer (4×6 vertical). Total generation time: 45 seconds.
Step 4 · T+15 min

Practice reviews — one text change requested

Agent emails Ashley preview thumbnails of all 3 assets with a direct Canva view link. Ashley replies: "These look great! On the flyer can you change the CTA to 'Ask us today' instead of 'Book now'?"

The agent extracts the revision (single CTA text change on the flyer), calls Canva API to update only the affected variable, and returns the updated asset in 90 seconds.
Step 5 · T+35 min

CD taps approve in 58 seconds

The Creative Director (Marcus) receives a Teams Adaptive Card notification on his phone with thumbnail previews of all 3 assets. He reviews them during a break between meetings.

Mobile-first review UI: thumbnail visible without scrolling, 3 actions visible: Approve All, Request Change, Escalate. Marcus taps Approve All. Total review time: 58 seconds. No desktop required.
Step 6 · T+45 min

Digital assets auto-deploy via Weave

On CD approval, the agent immediately exports the email header as PNG and TV slide as MP4 loop, schedules the Weave patient email campaign for the April 1 send date, and confirms deployment to Ashley.

Ashley receives: "Your April whitening special email campaign is scheduled to send to 847 patients on April 1. TV slide sent to your waiting room display. Flyer is ready for print — want me to order it?"
Step 7 · T+47 min

Print order auto-submitted to Gelato

Ashley replies "Yes, 50 copies please." The agent submits the print order to Gelato API: 50× 4×6 flyers, double-sided, standard stock, delivery to Towne Lake Dental's address on file.

Gelato confirms order number and 3–4 day delivery estimate. Agent forwards confirmation to Ashley. Total elapsed time from first email to fully deployed + print ordered: 52 minutes.
Step 8 · End of Week

Studio reviews weekly analytics dashboard

Marcus opens the Studio Analytics Dashboard on Friday afternoon. This week: 23 requests processed, 19 auto-fulfilled (83%), avg turnaround 48 minutes, medium challenge conversion 38% to digital-only.

Dashboard shows 4 practices with brand health scores below 40 (needs asset lift), 2 practices inactive for 90+ days, and print volume down 31% vs. 8-week baseline — on track toward the 34% reduction target.
System Architecture

8 Components — From Intake to Delivery

The system routes requests across email, Teams, and Slack into a single agent core, then fans out to design generation, digital deployment, print fulfillment, and analytics.

8 System Components

C1

Intake Channel Adapters

Email (M365 Graph API webhook), Teams (Bot Framework SDK v4), Slack Phase 2. Normalizes all inputs to a standard brief schema.

C2

Studio Agent Core

Claude API (claude-sonnet-4-6) as intelligence layer. Manages conversation state, brief completeness check, medium challenge logic, and revision loops.

C3

Practice Profile + DAM

PostgreSQL practice profiles with brand health scoring (0–100). Canva Enterprise brand kits as authoritative DAM. Asset gap detection and proactive outreach.

C4

Design Generation Engine

Canva Connect API for template variable injection and asset export. BullMQ + Redis for async generation jobs. 12-template library in Phase 1.

C5

Approval Workflow

Practice review loop via email reply-parsing. CD review via Teams Adaptive Card (mobile-first, tap-to-approve). 2-hour SLA with named backup.

C6

Digital Deployment

Weave API for patient email campaign scheduling. Phase 2: TV display integrations, social scheduling. Asset delivery confirmation back to practice.

C7

Print Fulfillment

Gelato API for print-on-demand. Nominated vendor replaces Staples. One-click order submission from agent conversation. Confirmation forwarded to practice.

C8

Studio Analytics Dashboard

React 18 + TanStack Query. KPIs: requests, auto-fulfill rate, turnaround, print volume. Brand health buckets. 90-day inactive flags. PostHog events.

Technical Decisions

7 Architectural Decision Records

Key technology and design choices with rationale. These decisions define the constraints for the development team and should not be reversed without documenting why.

ADR 1Claude API as Agent Intelligence Layer+

Decision: Use Claude API (claude-sonnet-4-6) as the AI backbone for all agent reasoning — brief extraction, medium challenge logic, revision handling, and brand quality assessment.

Rationale: claude-sonnet-4-6 provides the best balance of reasoning quality, instruction-following, and response latency for a real-time conversational agent. Structured JSON output mode enables reliable brief extraction without brittle regex parsing. Context window is sufficient for multi-turn creative conversations.

Consequence: Build a thin prompt management layer to version and A/B test system prompts without code deployments. Monitor token costs per request — target ≤$0.02/request at scale.

ADR 2Microsoft Bot Framework SDK v4 for Teams Integration+

Decision: Use Microsoft Bot Framework SDK v4 for Teams channel integration, with Adaptive Cards for the CD mobile review experience.

Rationale: SGA operates in Microsoft 365 — Teams is the existing communication platform for the CD and studio team. Bot Framework is the standard Microsoft-supported SDK for Teams bot development. Adaptive Cards provide a native Teams UI without requiring a separate mobile app, enabling the tap-to-approve CD review experience on any device.

Consequence: Bot registration required in Azure Bot Service. Teams app manifest must be deployed to the SGA tenant. Bot Framework version lifecycle must be tracked — v4 is current stable.

ADR 3Canva Connect API for Brand Kit Management and Design Generation+

Decision: Use Canva Connect API (REST) as the design generation engine. Canva Enterprise brand kits serve as the authoritative DAM per practice.

Rationale: The studio already uses Canva Enterprise with established brand kits and a template library. The agent wraps existing infrastructure with API access rather than replacing it. Canva's template variable system allows programmatic content injection without requiring design skills in the agent layer.

Consequence: Spike required (SP-1): Canva Connect API template variable limits must be validated before committing to the 12-template library. If variable limits are insufficient, fallback to Canva's image export API with pre-rendered templates. Canva Enterprise plan must include Connect API access — confirm licensing before build.

ADR 4Gelato as Nominated Print Vendor via API+

Decision: Replace Staples and ad-hoc print vendors with Gelato as the nominated print-on-demand vendor, integrated via Gelato's REST API.

Rationale: Gelato provides API-first print fulfillment with global production nodes, predictable pricing, and faster SLA than Staples. API integration enables the agent to submit, track, and confirm print orders conversationally — no manual portal logins, no PO process. Volume consolidation across 160+ practices creates leverage for pricing negotiation.

Consequence: Gelato account setup and API credentials required before Phase 1 launch. Paper stock, finish options, and pricing tiers must be configured. Practices with existing preferred print vendor relationships need a clear transition communication plan.

ADR 5BullMQ + Redis for Async Design Generation Jobs+

Decision: Use BullMQ with Redis as the async job queue for Canva API generation requests, rather than awaiting them inline in the agent conversation loop.

Rationale: Canva API generation takes 15–60 seconds. Holding an open HTTP connection for this duration is fragile and prevents the agent from handling other requests. BullMQ provides reliable job queuing, retry logic on API failures, and progress callbacks that allow the agent to send "generating now..." acknowledgment and then follow up when complete.

Consequence: Redis instance required (Azure Cache for Redis). BullMQ worker process must be deployed alongside the main API server. Job status must be persisted so the agent can resume a conversation after a server restart.

ADR 6PostgreSQL + Drizzle ORM for Practice Profile Data+

Decision: Use PostgreSQL (Azure Database for PostgreSQL) with Drizzle ORM for all structured data: practice profiles, brand health scores, request history, conversation logs, and analytics events.

Rationale: Practice profile data is relational (practice → brand kit → requests → assets → analytics events). PostgreSQL handles this naturally with strong ACID guarantees. Drizzle ORM provides type-safe schema definitions in TypeScript with lightweight overhead vs. Prisma. Azure PostgreSQL integrates cleanly with the broader Azure infrastructure plan.

Consequence: Migration tooling via Drizzle Kit. Schema versioning required from day one — practice profile schema will evolve as Phase 2 adds PMS integration fields. Azure PostgreSQL connection string stored in Azure Key Vault.

ADR 7Azure Infrastructure for Microsoft Ecosystem Alignment+

Decision: Deploy all infrastructure on Azure: App Service (API server), Bot Service (Teams bot), Static Web Apps (analytics dashboard), Cache for Redis, Database for PostgreSQL, Key Vault (secrets), Application Insights (monitoring).

Rationale: SGA operates in Microsoft 365 (Teams, Exchange, SharePoint). Azure provides native integration with M365 for authentication (Azure AD / MSAL), Teams bot registration, and Microsoft Graph API access. A single cloud vendor reduces operational overhead and simplifies the network security model for M365 service-to-service calls.

Consequence: Azure subscription with appropriate spending limits required. Infrastructure-as-code (Bicep or Terraform) required from day one — no manual portal deployments. Cost monitoring dashboard recommended given multi-service architecture.
