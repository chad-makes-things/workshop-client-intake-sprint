Monday · Frame Goal + Sprint Questions

Long-term goal: In 18 months, every SGA-affiliated practice — all 160+ locations — can request, receive, and deploy on-brand creative assets in under 48 hours through an AI-agent-powered studio, without emailing a designer for templated work. The studio team shifts from reactive order-takers to proactive brand stewards.

1
Can a practice manager with zero design training successfully submit a complete, usable creative brief through a chatbot — and get back an on-brand asset without calling a human?
Risk: If no, the bottleneck stays with the studio team. Scale is impossible.
2
Can AI-generated designs in Canva consistently meet SGA brand standards, with only a single human quality gate standing between generation and delivery?
Risk: If no, human review load grows instead of shrinking. The model breaks.
3
Can we reduce discretionary print volume by 30–40% by building a digital alternatives challenge directly into the intake flow — before a practice commits to print?
Risk: If no, we automate a costly, low-ROI behavior and miss the medium-shift opportunity.
4
Can we create a centralized digital asset management system seeded with brand foundation packages for underserved practices — preventing requests from failing because assets don't exist?
5
Can we consolidate print procurement to a single curated vendor channel that practices order from directly — with the studio only setting up and auditing, not touching each order?
Monday · Frame Target Selection
Confirmed target · Score 24/25
The intake and routing moment — Steps 2–4 of the problem map
The moment a practice manager submits a creative request and the system determines what they actually need, whether print is the right medium, whether assets exist to fulfill the request, and which workflow to trigger. This is the only part of the system with no current solution. Fixing it cascades forward — bad intake produces bad briefs, wrong designs, vendor failures, and late deliveries.
✓ Highest-leverage point ✓ Fully greenfield ✓ Where AI has the most natural role ✓ Where the medium challenge lives

Key Learnings
Learning 1

The problem was never capacity

The constraint is friction density per request — too many touches, too many handoffs, too many days. The actual request volume is manageable for a two-person team if the workflow is right.

Learning 2

The medium challenge is the highest-leverage feature

Every solution can automate design. Only Solution A challenges the medium before print is committed to. Over 160 locations, practices redirected from print to digital compounds fast.

Learning 3

Email creates the audit trail

Every request, brief, revision, and approval is threaded in email. For a multi-location DSO with brand governance needs, that documentation is inherently valuable — and expected by legacy practices.

Learning 4

The asset backfill is the critical path

~30-40% of practices lack adequate brand assets for automated generation. Brand kit backfill runs in parallel with the agent build — both must land at the same time for scale.

Learning 5

CD review is a feature, not a bottleneck

Designed correctly (practice confirms content first, CD judges brand quality only), the 60-second gate is achievable. It becomes a quality signal and governance mechanism, not a delay.

Learning 6

Vendor consolidation is a workflow problem

The nominated vendor wins through convenience, not mandate. Making the one-click Gelato path so much easier than calling Staples means practices choose it without being told to.

Architectural Decision Records

ADR 1
Email as the third intake channel (studio@sgadental.com)
▼
Teams and Slack reach Gen4 well — but legacy SGA and MODIS practices may not be on those platforms. Email is the universal access layer. Every practice manager has it. A dedicated studio address is also legible as a "real thing" — more so than a bot mention in a channel they don't regularly use. It also creates a searchable audit trail of every request for brand governance.
ADR 2
Claude API for the agent core, not a decision-tree chatbot
▼
A decision tree handles "I need a business card" correctly. It breaks on "We're doing a Lunch & Learn at the perio office next week and need some collateral." The dental practice context is complex — the agent needs to understand what a liner special is, what a PRD does, why a doctor referral card differs from a patient-facing flyer. Claude's language understanding handles edge cases that would require hundreds of decision tree branches.
ADR 3
Canva Enterprise as both DAM and design tool
▼
A dedicated DAM (Brandfolder, Bynder) plus a separate design tool (Adobe CC, Figma) adds a file-transfer step, two vendor relationships, and a more complex API integration. Canva Enterprise handles both functions with a single API. The Brand Kit feature provides per-practice asset storage with brand locking. The Connect API allows programmatic template variable population and export.
ADR 4
Template library starts at 8 for Phase 1 — studio-admin expandable
▼
The critical design decision: adding a new template is a studio designer task in Canva — not an engineering sprint. This removes the bottleneck that would otherwise slow template expansion. The studio team can identify gaps from analytics (what's being escalated to human designers?) and add templates without a software release cycle.
ADR 5
Practice preview before CD review (not after)
▼
Standard agency workflow: design → client review → revisions → internal sign-off. We reversed the middle two steps. If the CD approves and the practice then requests a content change, the CD must re-review. By having the practice confirm content first, the CD's review is exclusively a brand quality judgment on a finalized asset. This reduces post-CD change requests from ~30% to under 5%.
ADR 6
Digital deployment integrations phased — TV slide manual in Phase 1
▼
The variety of in-office display systems across 160 practices makes a single-API TV integration impossible in Phase 1. Rather than block the launch, Phase 1 delivers TV slides as files with upload instructions. Phase 2 maps the most common display systems in the Gen4 network and builds the integration for those. This ships value in Phase 1 without waiting for perfect automation.
ADR 7
Nominated vendor enforced through workflow — not policy
▼
Policy-based vendor consolidation fails because practice managers under time pressure use whatever is fastest — Staples, a local shop, the office color printer. Workflow-based enforcement works because the only "Order Print" button routes to Gelato. Practices can download the PDF and use any vendor — but they get no studio support for that path. The nominated vendor path is so much easier that practices choose it without being told to.
