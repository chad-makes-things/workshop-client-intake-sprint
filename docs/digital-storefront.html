User Journey

From current state to fully managed storefront

How a practice experiences the Practice Concierge — from silent discovery through ongoing change management.

Step 1 — Before Contact

Discovery & Pre-Population

System silently harvests public data — websites, GMB, social profiles, reviews, directories. Builds a Practice Profile Draft with confidence scores per field.

Uses LLM-assisted extraction to parse any website structure. The "we already know about you" moment. Quality here determines first-impression impact.
Step 2 — Invitation

Enrollment Invitation

Batches of 20-30 practices receive personalized invitations with a preview of what the system already knows about them. "Meet your concierge and get upgraded in 15 minutes."

Must feel like a gift, not a mandate. The preview data demonstrates value before asking for anything.
Step 3 — Verification

Concierge Welcome & Data Verification

Concierge presents harvested data. Practice corrects in 3-4 minutes: "Dr. Miller left, add Sarah." Every correction triggers downstream cascades automatically.

Sage + Caregiver tone — knowledgeable, warm, never bureaucratic. "Got it — I'll remove Dr. Miller from everything."
Step 4 — Generation

Storefront Generation

2-3 specialty-optimized templates populated with real data. Concierge recommends with reasoning: "Perio sites convert 23% better with referral-optimized layouts."

Guardrailed decisions — every option is SGA-approved. If no response in 3 weeks, auto-deploys the recommendation.
Step 5 — Review

Full Storefront Approval

Complete package: website, GMB, social profiles, lead capture. HQ creative reviews in parallel (24-hour QC window). Practice approves, everything deploys.

99% pass QC without issue. Real-time changes handled in-conversation.
Step 6 — Go Live

Activation

Website, GMB, social, forms — all live. Concierge sends 1-week follow-up: "Your new site had X visitors. Y appointment requests."

First data delivery reinforces value and builds the check-in habit.
Step 7 — The Moment of Truth

Change Management

A doctor leaves. Office manager tells the concierge. It responds with a complete action plan across all 7+ touchpoints, scheduled for the departure date.

This is where long-term value is proven. Proactive suggestions demonstrate opinion and thoroughness.
Step 8 — Safety Net

Blitz Deploy

After 30 days, non-engaged practices get default storefronts auto-deployed from scraped data. "I went ahead and launched your upgrade — customize anytime."

100% coverage regardless of engagement. Dashboard shows "auto-deployed" vs "practice-customized" tiers.
System Architecture

Six components, one orchestration layer

Event-driven architecture with a single source of truth for all practice data.

Practice Data Hub

Single source of truth. One canonical record per practice. Change here and it propagates everywhere. PostgreSQL + Redis cache for fast agent context loading.

Data Harvester

Pre-populates practice profiles from public data using LLM-assisted extraction. Runs before any practice interaction to build the "we already know you" moment.

Concierge Agent

Per-practice AI concierge. Handles enrollment, change management, content approval, Q&A. Opinionated — always recommends with reasoning. Sage + Caregiver personality.

Template Engine

5 specialty-specific website templates generated from structured data. One template update re-generates all sites using it. Batch operations are native.

Channel Sync Engine

Event-driven sync to GMB, social, directories, CRM, and reviews. Changes fire in minutes, not nightly batches. Retry with exponential backoff.

Network Dashboard

Executive + HQ views. Automated health scores, alerts, QC queue, batch operations. Zero manual data entry — every metric pulls from live sources.

Architecture Decisions

The reasoning behind critical choices

Each decision record documents what was decided, why, and the trade-offs accepted.

ADR-1: One Agent Per Practice (not shared) +

Decision: Deploy one logical agent instance per practice — shared model, practice-specific context.

Why: Per-practice agents feel personal and eliminate context-switching errors. The LLM is shared; only the system prompt and context window differ. At 260+ practices, overhead is context storage, not compute.

Trade-off: Practice data must be efficiently loadable into each agent's context on every interaction. Requires a practice context service.

ADR-2: Generated Sites (not CMS-managed) +

Decision: Generate static/JAMstack sites from templates + data, not CMS-managed dynamic sites.

Why: 260 WordPress instances is a maintenance nightmare. Generated sites are faster, cheaper, more secure, and easier to batch-update. One template change re-generates all sites.

Trade-off: Content updates require re-generation and re-deployment. Pipeline must produce live updates within minutes.

ADR-3: Event-Driven Sync (not nightly batch) +

Decision: Event-driven synchronization with nightly batch as safety net.

Why: "Dr. Johnson left" can't wait for a nightly batch — the website would show a departed doctor for up to 24 hours. Events fire within minutes. Batch catches anything the event system misses.

Trade-off: Requires a reliable message queue and graceful API rate limit handling with retry logic.

ADR-4: Blitz Deploy Safety Net +

Decision: Auto-deploy default storefronts for non-engaged practices after 30 days.

Why: A clean generic storefront from scraped data is better than a broken or nonexistent one. Ensures 100% network coverage regardless of practice engagement.

Trade-off: Blitz-deployed sites may have inaccuracies. Flagged as "auto-deployed — unverified" in the dashboard.

ADR-5: Opinionated Agent with Reasoning +

Decision: The Concierge always recommends with evidence-based reasoning, never just presents options.

Why: SGA's brand is Sage + Caregiver. An agent that says "here are 5 options" adds no value. "I recommend Template A because perio sites convert 23% better with referral-optimized layouts" reduces decision fatigue and demonstrates expertise.

Trade-off: Knowledge base must include conversion data and industry benchmarks. Wrong recommendations erode trust permanently.

Key Learnings

What the sprint revealed

Insights that emerged through structured divergent-convergent thinking.

Insight 01

The onboarding problem is really a data problem

Multi-month setup time isn't because building websites is hard. It's because getting the data (passwords, brand assets, team info) is hard. Pre-populate from public sources, verify in 15 minutes.

Insight 02

The agent IS the product, not a feature

The concierge should be the ONLY interface practices see. Everything else is backend infrastructure. One thing to talk to, not five tools to learn.

Insight 03

Opinionated by default, customizable by exception

The system comes in with strong opinions backed by data. 90% get a great storefront without lifting a finger. 10% get guardrailed choices. Nobody gets a blank canvas.

Insight 04

Blitz deploy prevents the adoption trap

If the system requires engagement to deliver value, non-engaged practices get nothing. Auto-deploy defaults ensure 100% coverage regardless of practice behavior.

Insight 05

Stage 2 is an extension, not a rebuild

The enrollment agent becomes the curation agent. The data hub becomes the curation data layer. Everything we build for onboarding feeds directly into 365-day operations.
