The Problem

75+ disconnected systems create a 30-day blind spot across the patient funnel at every practice.

Today

  • 75 individual tech platforms across practices
  • 30+ days post month-end to assess business health
  • Practice staff juggle 10+ tools in ~20 min/day
  • No standard CRM or lead capture layer
  • Phone answer rates vary 50-90% — nobody knows until it's too late
  • No-shows discovered after the fact, never prevented
  • Executive visibility is anecdotal, not data-driven
  • Problems found retroactively — can't be proactive

With Funnel OS

  • 15 core platforms unified through one integration layer
  • Real-time funnel visibility — no month-end wait
  • One 20-min agenda replaces 10 separate tools
  • Unified funnel data model from lead to production
  • AI monitors every call metric across every practice
  • At-risk appointments flagged 24h before and auto-confirmed
  • Executive dashboard: network → region → practice drill-down
  • AI spots problems and generates interventions in real-time

Why Rejected Solutions Were Rejected

The Nerve Center (26/35)

Best practice adoption, but executive visibility depends on practice usage. Bottom-up approach is slower to network-wide coverage and doesn't solve the data integration problem architecturally.

The Data Mesh (24/35)

Cleanest integration pattern, but over-engineered for current needs. Most dental platforms don't have event APIs — "event emitters" would really be pollers in disguise. Slow to deliver user-facing value.

Dental Command Center (22/35)

Fastest to executive dashboard via BI tool, but fragments into 3-4 separate systems. BI platforms display data, they don't act. Practice Nerve Center would be completely disconnected.

The SGA Intranet (21/35)

Most complete UX vision — replaces the concept of "tools." But 9-12 months to meaningful value and the biggest practice adoption hurdle. Embedding external tools is technically fragile.

Key Learnings

The problem is integration, not intelligence

If you solve the integration layer, the AI and dashboard layers are comparatively straightforward. Data plumbing is 60% of the work and 80% of the risk.

The funnel model IS the product

The 5-stage funnel (Lead → Connected → Booked → Showed → Produced) is the data schema, the AI's framework, and the staff's action agenda — all in one.

Don't show practice staff data — show them actions

The "20 minutes, 10 tools" constraint forced the design toward an "agenda" pattern. 3 cards, each tappable, each with one action.

Concierge + Nerve Center must converge

Building two practice-facing tools creates the exact fragmentation problem we're solving. One app, modular tabs — the most important architectural decision.

Architecture Decisions

Key decisions and their rationale.

ADR 1: Unified Platform vs. Best-of-Breed Assembly

▶
Decision: Build a unified platform rather than assembling BI tool + warehouse + AI + practice app as separate systems.

Rationale: System fragmentation is the current problem. Building a fragmented solution perpetuates it. One platform costs more upfront but ensures data consistency, reduces maintenance, and enables AI to operate across all layers.

Tradeoff: Slower time to first value — mitigated by BI tool as Stage 1 dashboard.

ADR 2: BI Tool as Stage 1 Dashboard (Buy Then Replace)

▶
Decision: Metabase/Superset as Stage 1 exec dashboard, replaced with custom React dashboard in Stage 3.

Rationale: Executives need data in 30 days. Custom dashboard takes 3-5 months. BI tool validates the data model and executive needs before investing in custom UI. It's disposable infrastructure.

Tradeoff: Stage 1 looks "generic." But executives see real, real-time data from day 30.

ADR 3: Per-Practice Baselines (Not Network Averages)

▶
Decision: Anomaly baselines per-practice from 90-day rolling history, not network averages.

Rationale: A 2-chair rural practice ≠ a 12-chair metro practice. Network averages create false positives and false negatives. Per-practice baselines ensure alerts are meaningful.

Tradeoff: 90 days before reliable baselines. Mitigation: cohort-based priors for new practices.

ADR 4: Nerve Center Extends Practice Concierge

▶
Decision: The Practice Nerve Center extends the Practice Concierge from the Storefront sprint — one app, modular tabs.

Rationale: Same users, same interaction model, same adoption challenge. Two separate tools defeats "replace 10 tools with 1." The Concierge handles marketing/brand, the Nerve Center handles operations/revenue.

Tradeoff: More complex app. Mitigation: separate modules, shared navigation and AI (like Gmail tabs).

ADR 5: Service-Type-Aware Automation

▶
Decision: All automation parameterized by service type — cleaning ≠ implant ≠ full-arch.

Rationale: Patient psychology, no-show risk, revenue, and appropriate tone vary dramatically. The revenue difference between a $150 cleaning no-show and a $30,000 implant no-show justifies differentiation.

Tradeoff: More templates, more rules. But contextual automation builds trust; generic automation erodes it.
